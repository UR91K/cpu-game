Overview

This document describes the design of the spatial audio engine for a 2D tile-map game rendered with wgpu. The engine approximates physically plausible room acoustics using runtime 2D path tracing on the CPU, decoupled from both the game simulation and audio mixing threads.

The map is static at runtime, which simplifies the acoustic problem: tile materials and topology never change, so path tracing results are stable between updates and can be updated lazily at 10–15 Hz without perceptible degradation.

Key Decision The static map was considered for offline bake / precomputed probe grids, but arbitrary geometry (spirals, mazes, irregular rooms) makes dense probe grids equivalent in cost to runtime path tracing with worse generality. Runtime path tracing on a dedicated CPU thread was chosen as simpler, more correct, and well within the available CPU budget (Ryzen 5 3600 at ~0.6% baseline utilisation).

Sound events travel from the game thread directly to the audio thread via a lock-free channel, bypassing the acoustic thread entirely. This means sounds play immediately — the acoustic thread only governs the treatment applied to already-playing sounds, not whether they play at all.

Stale acoustic params on the audio thread (up to ~100ms behind) are imperceptible. Acoustic parameters change only as the listener or source moves through space — at normal movement speeds the perceptible change between 10Hz updates is negligible.

Acoustic Path Tracing

The core acoustic calculation is 2D path tracing on the tile grid — conceptually analogous to light transport but with different physical rules. The tile map provides free axis-aligned wall normals, uniform wall thickness, and discrete material lookup, all of which simplify the implementation significantly.

DDA Direct Path

For each active sound source, a DDA ray is cast from the source position to the listener. Each wall tile traversed accumulates absorption and low-pass mix. If the direct path is unoccluded and transmission energy remains above threshold, no further tracing is needed.

fn trace_direct(map: &TileMap, from: Vec2, to: Vec2) -> DirectPath {
    let mut transmission = 1.0f32;
    let mut lpf_mix      = 0.0f32;

    for tile in dda(from, to) {
        if let Tile::Wall(mat) = map.get(tile) {
            transmission *= mat.transmission;
            lpf_mix       = (lpf_mix + mat.transmission_lpf).min(1.0);
            if transmission < 0.01 { break; } // below audibility threshold
        }
    }

    DirectPath { transmission, lpf_mix }
}

BFS Indirect Path Routing

When the direct DDA path is fully occluded, or the source is in geometrically complex space (spiral room, maze), a BFS is run on the open-tile graph weighted by inverse local width. This finds the path sound would naturally travel — through corridors, around corners, via the least resistive route — regardless of map shape.

The narrowest point on the found path becomes the acoustic bottleneck — the re-emission point from which the sound appears to arrive at the listener. This correctly reproduces the "focused reverb from a doorway" effect without requiring precomputed portal graphs.

fn find_acoustic_bottleneck(
    map:  &TileMap,
    from: Vec2,
    to:   Vec2,
) -> Option<Vec2> {
    // walk BFS path; find tile with minimum open neighbour count
    bfs_path(map, from, to)?
        .into_iter()
        .min_by_key(|&tile| count_open_neighbors(map, tile))
        .map(|tile| tile.center())
}

Early Reflection Bounce Rays

Up to 2 specular bounce rays are cast from the source to find indirect reflection paths. Wall normals are always axis-aligned on a tile map, so the reflection direction is trivial to compute. Material diffusion determines whether a bounce produces a discrete early echo or feeds energy into the diffuse reverb tail.

Beyond 2 bounces, remaining energy is folded into the parametric reverb tail rather than traced explicitly. This keeps the per-source ray count bounded.

// specular: discrete echo at known delay + direction
// diffuse:  energy added to reverb tail, no distinct echo
enum ReflectionKind { Specular, Diffuse }

struct EarlyReflection {
    delay:     f32,   // path length / SPEED_OF_SOUND
    energy:    f32,   // compounded absorption
    direction: Vec2,  // arrival direction at listener
    kind:      ReflectionKind,
}

Path Statistics → Reverb Parameters

The BFS path through the map is analysed to derive parametric reverb parameters. Mean path width drives reverb time; width variance drives diffusion character. These work correctly for arbitrary geometry — a tight maze produces short, directional reverb; a large irregular cavern produces long, diffuse reverb.

fn path_to_reverb(path: &AcousticPath) -> ReverbParams {
    let mean_width = path.mean_segment_width();
    let variance   = path.segment_width_variance();

    ReverbParams {
        // narrow path → short decay; wide path → long decay
        reverb_time: path.total_length * (1.0 / mean_width) * 0.05,

        // low variance (uniform width) → diffuse reverb
        // high variance (corridor → room) → directional character
        diffusion: 1.0 - variance.sqrt().min(1.0),

        pre_delay: path.total_length / SPEED_OF_SOUND,
    }
}

Physical Phenomena

Assessment of each acoustic phenomenon for relevance in the tile-map model.

Phenomenon Priority Approach
Spreading / divergence Must Inverse square falloff on direct path length. Modified by path transmission for indirect paths.
Absorption Must Per-material coefficient accumulated during DDA. Approximated with two frequency bands: full-range and low-passed. More wall hits → more LPF blend.
Transmission Must Per-material transmission fraction. All walls are one tile thick so no thickness modelling is needed — material is the only variable. Multiple walls compound multiplicatively.
Reflection Must Up to 2 specular bounce rays. Axis-aligned tile walls give free wall normals. Material diffusion flag routes energy to discrete echo vs reverb tail.
Diffusion Must Per-material flag (smooth → specular reflection, rough → diffuse). Diffuse reflections feed the reverb tail rather than producing discrete echoes.
Diffraction Should Corner tiles (open tile adjacent to wall edge) treated as weak secondary emitters. Prevents unnatural hard cutoff when listener rounds a corner.
Interference Ignore* Full wave interference requires phase simulation. Exception: flutter echo in narrow parallel corridors approximated via path width variance feeding a feedback delay.
Refraction Ignore Requires medium density / temperature gradients. Not relevant to tile-map geometry. A per-medium speed-of-sound constant is sufficient if underwater tiles exist.

Acoustic Params

The output of one path trace cycle for a single source. Written by the acoustic thread, read by the audio thread. Describes "what treatment would be applied to any sound emitted from this entity's current position."

struct AcousticParams {
    /// final volume after transmission + distance falloff
    volume:       f32,
    /// stereo pan derived from bottleneck/source position relative to listener
    pan:          f32,
    /// low-pass mix (0 = dry, 1 = fully low-passed)
    lpf:          f32,
    /// seconds before reverb tail begins (path length / speed of sound)
    pre_delay:    f32,
    /// RT60-equivalent decay time derived from path width statistics
    reverb_time:  f32,
    /// how much energy feeds into the diffuse reverb tail (0–1)
    reverb_send:  f32,
    /// diffusion character: 1 = fully diffuse, 0 = directional
    diffusion:    f32,
    /// up to 8 early discrete reflections
    early:        [EarlyReflection; 8],
    /// world position from which sound appears to arrive (bottleneck)
    emission_point: Vec2,
}

State Model

The acoustic update follows the same pure-function pattern as the game simulation. update_acoustics takes the current state and produces a new one — no mutation, easy to reason about, compatible with the existing architecture.

pub fn update_acoustics(
    state:  &AcousticState,
    game:   &GameState,
    level:  &Level,
) -> AcousticState {
    let mut next = state.clone();
    next.listener = extract_listener(game);
    next.sources  = collect_sound_sources(game);
    next.params   = next.sources.iter()
        .map(|(&id, src)| {
            let params = trace_acoustic_path(level, src, &next.listener);
            (id, params)
        })
        .collect();
    next
}

Non-Goals

The following are explicitly out of scope for this implementation:

ItemReason
GPU compute for path tracing Workload is too small to saturate a GPU. Dispatch overhead exceeds computation cost. CPU has ample headroom.
Offline acoustic bake Probe grids dense enough for arbitrary geometry (spirals, mazes) offer no meaningful advantage over runtime tracing. Adds pipeline complexity for no perceptible gain.
HRTF / binaural rendering Requires head-related transfer function datasets. Stereo pan via rodio's built-in .pan() is sufficient for 2D gameplay.
Wave simulation / full interference Requires per-frequency phase tracking. Computationally prohibitive. Flutter echo approximation via path statistics covers the perceptually significant case.
Frequency-dependent absorption beyond 2 bands Diminishing perceptual returns beyond full-range + low-passed blend.
More than 2 specular bounce rays Energy beyond 2 bounces is low enough to fold into the diffuse reverb tail without perceptible difference.
