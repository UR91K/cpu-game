# External Crate References

When working with or troubleshooting issues related to **wgpu**, **winit**, or **librashader** crates, and Aseprite source code, always reference the original source code and documentation available in the `external-references/` directory.

## Available References

- `external-references/wgpu/` - wgpu graphics library source and docs
- `external-references/winit/` - winit windowing library source and docs  
- `external-references/librashader/` - librashader shader library source and docs
- `external-references/aseprite/` - Aseprite application source code

## When to Use

- Investigating API usage or behavior
- Debugging integration issues
- Understanding implementation details
- Resolving compilation errors
- Checking for breaking changes or deprecated features
- Verifying correct usage patterns

## Best Practices

1. Check the local reference first before searching online documentation
2. Look for examples and tests in the reference directories
3. Review actual implementation when behavior is unclear
4. Cross-reference with project usage to ensure compatibility
5. Note version-specific differences if applicable
